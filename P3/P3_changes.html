Descriptions of the pages:

There have been many changes to the entire website. As a result of our expanded knowledge, the content and structure of the website was modified. The modifications to the specific pages are listed below. If the page is not listed, no changes have been made since the original plan.

Change to php:

We decided to use php includes to minimize duplicate code across the website. Each page now uses php include to add the following content:

  • head
  • banner
  • navbar
  • footer
Using php-includes has taught a lot about code efficiency, and seems to be working great.

We have two pages using forms, and we were a bit uncertain as to what to do with the data from said forms, in 07contact.php, and 08donate.php. We ended up with setting up two separate php pages to receive the data, which seems to work quite well. Since no databases were allowed, the data will not be stored, but temporarily received, and most of all, it shows that the forms work as intended.

Global Layout

Banner, navbar, and footer all got individual changes that span across all pages.

The banner was slightly changed, with the removal of the background padding around the text. We felt that what we ended up with going with was more simplistic, and fit the overall style of our site better. The pixel values of the banner remains close to the same, with a deviation from 132px width on the cats on each side, to 133px width. This was to achieve better scaling within the pictures on the sides.

The navbar was originally intended to stretch across the whole page, but it felt a bit stretched out in the middle, so we decided to lower the spacing on each side, thus bring each link closer together. We also removed the borders in order to keep a more simplistic style. This lead to the drop-down on "Adopt" and "About" not being centered, but instead we focused on having the same left-margin on the drop-down-items, thus adding a bit of padding to the right. We agreed that this looks better, and are satisfied with the functionality. We also decided to add a h2 to each page, indicating clearly which page the user was on. This was initially thought to be done by having the navbar-tab highlighted, but this would have lead to "Adopt" and "About" being dropped down while browsing their subpages, which we felt was a suboptimal solution.

When we started the project, and wrote P2_requirements, we initially thought we would be using javascript for the navigation bar. At that stage we were honestly not really sure how to implement javascript efficiently, and thought it seemed like a reasonable choice to have it for the navbar. Upon creation, we found that the navbar worked great without javascript, and since the focus of the website shouldn't be smartphones, our minimum resolution did not require changes to the navbar via javascript. We thus decided to skip it entirely for this section, and instead made up for it in other pages, where we felt it would be put to better use.

As this is a website for a ficional client, we did not want to put a valid email at the bottom of each page, and thus left a tag instead.

02ourCats.php

As mentioned previously, we weren't really sure what we could expect from our javascript-implementations. 02ourCats.php is one of the pages that had the most changes to them during the creation process. We ended up with going with a very simplistic html, which consists of 5 unique lines. The rest is filled inn via javascript, and we are very happy with this solution.

The javascript 02ourCats.js is one of our most changed files. We initially had a lot of content in the html, but were able to remove this, and implement it all via javascript instead. An array is created, and then the CatLady will manually add objects below, with catInfo: "Cat Name", and catSrc: "link to image of Cat Name". All these objects are manually pushed into the catArray, which makes it easy to remove a single cat from the list, in case it was adopted. The function catSlideShow then pushes values from the array to the DOM, and loops with a three second interval between each new cat. We are very happy with the way this script turned out.

07contact.php

The contact page has some minor changes to the text in the left-hand side of the body, filled with more useful information. The contact form was altered to have a border to clearly display where it begins and ends, and style was simplified. We also added a select-drop-down-list, so the user, and Cattius, can easily determine the topic of the exchange.

Changes to our time plan:

The members of this group had other projects with deadlines which requied their attention. Therefore, some of the deadlines for this projects had to be postponed. Specific dates for these were not set, however. This affected the project workload, since many things were done at the same time, at the end of the project. Nevertheless, we managed to still finish the project in time for testing it before the delivery.
